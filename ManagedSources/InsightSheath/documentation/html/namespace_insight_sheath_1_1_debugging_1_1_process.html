◆ DebugModeType

enum InsightSheath.Debugging.Process.DebugModeType

Flags to indicate to InsightProcess if you want the worker thread or not.

Enumerator
NoWorkerThread 

Tells Insight's native DLL to Not Spawn any worker threads for debug handling. If you don't implement a debug loop, you will never see your process show up. Irrelevant if the process is not being debugged. You're going to want the WorkerThread as development features in this thread are not exported yet for use without the worker thread.

WorkerThread 

When Spawning a process for debugger, InsightApi.DLL implements a debug loop and offloads the loop to a worker thread. Put a call to InsightProcess.PulseDebugEventThead to continue after handling your debug event on a regular basis. Skipping that means, your debugged process won't continue after the first event.
